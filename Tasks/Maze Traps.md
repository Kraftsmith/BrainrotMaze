# Ловушки в лабиринтах

## Цель

По дизайну ([Labyrinths.md § 4](../Game Design/Labyrinths.md)) с уровня 2 (Редкий) и выше в лабиринтах появляются ловушки. **Эффект при срабатывании:** игрок роняет брейнрота, которого нёс — брейнрот падает на пол, его можно подобрать снова (в т.ч. другому игроку).

Ловушки добавляют риск-награду в основной цикл: чем глубже лабиринт, тем больше шанс уронить ценного брейнрота на пути обратно.

---

## Что уже есть в проекте

**`ServerScriptService.WallTrapScript`** и **`ServerScriptService.WallMovementScript`** — два скрипта, **которые НЕ переиспользуем** по следующим причинам:

1. Оба захардкожены под один трэп через `workspace.wall1` / `wall2` / `act` / `active` / `stop` (top-level). Не масштабируется на несколько ловушек.
2. **WallTrapScript ставит `humanoid.Health = 0`** на касание стены — убивает игрока. По дизайну ловушка должна **ронять брейнрота**, не убивать.
3. Оба подписаны на одни и те же `Touched` события → race condition, обе функции запускают движение стен одновременно.
4. В Workspace нет ни одного из ожидаемых частей (`wall1`, `act`, `stop`) — скрипты сейчас ничего не делают, мёртвый код.

**Действие:** оба скрипта отключить (`Disabled = true`) в рамках этой задачи. Старая логика "стены сдвигаются" при желании можно реанимировать как один из типов trap'ов после рефакторинга, но не наследуем код.

---

## Функциональные требования

1. **Эффект ловушки** = `BrainrotPickup.dropTop(player)` × N (где N = количество несомых брейнротов, обычно 1–2). После срабатывания все несомые брейнроты падают перед игроком на пол (Anchored = true), снова доступны для пикапа.
2. **Игрок не умирает** от ловушки. Только теряет добычу. (Опционально: визуальный шейк / звук / красная вспышка экрана.)
3. **Дебаунс** — после срабатывания ловушки на конкретного игрока, игнорировать его повторные касания N секунд (например, 1.5с). Иначе пробежка через зону шипов уронит, тут же подберёт назад в той же зоне, опять уронит — крайне фрустрирующе.
4. **Типы ловушек** (по дизайну):
   - **Шипы из пола** — выдвигаются по периодическому таймеру, нужно проходить когда они опущены.
   - **Огненные струи из стен** — периодически бьют поперёк коридора.
5. **Прогрессия по тиру лабиринта**: чем выше тир — тем больше плотность ловушек, быстрее срабатывание / короче окно безопасности.
6. **Ловушки только в лабиринтах**, не в лобби и не на базах.

## Решённые вопросы

1. **Интеграция с пикап-системой** — **BindableEvent `TrapHit(player)`** (слабое связывание). Создать `ServerStorage.BrainrotEvents` с `TrapHit` (плюс место для остальных событий из задачи [Явное событие BrainrotPickedUp](../.kanbn/tasks/Явное-событие-brainrot-picked-up.md)).
2. **Архитектура trap'а** — **Model** с тегом `MazeTrap`. Внутри хитбокс-Part(ы) с тегом `TrapHitbox` + визуалы отдельно.
3. **Damage-источник** — оба: шипы (CanTouch on/off) **и** огонь (Beam.Enabled on/off). Хитбокс активен только в окно срабатывания.
4. **Тир лабиринта** — атрибут `Tier` на корневой модели лабиринта. Common maze=1, Rare=2, Epic=3, Legendary=4, Rainbow=5, secret(soon)=6.
5. **Расстановка** — **автогенерация по raycast** (тот же подход, что и для спавн-пэдов). Скрипт сгенерит N trap-объектов на полу выбранного лабиринта.
6. **Визуалы** — **простые Roblox-параметры**: для шипов — Wedge-парты, для огня — `Beam` + `ParticleEmitter`. Без Toolbox-импортов.
7. **Эффекты помимо drop** — **drop + звук**. Без screen shake, без partikл на игроке.

## Рекомендуемая архитектура

- **Тег `MazeTrap`** на корневой Model каждой ловушки.
- На модели — атрибуты:
  - `TrapType` ∈ `{"Spikes", "FireJet"}`
  - `Period` (секунды между срабатываниями)
  - `ActiveDuration` (как долго опасна за цикл)
  - `Phase` (опционально, чтобы соседние ловушки не палили в такт)
- Внутри модели:
  - Один или несколько `BasePart` с тегом `TrapHitbox` — только их `Touched` слушаем
  - Визуальные меши (шипы, эмиттер огня, Beam) — отдельно
- **Один скрипт `MazeTrapsServer`** (Server, ServerScriptService):
  - При старте + `GetInstanceAddedSignal("MazeTrap")` — подвязывает каждую ловушку к её циклу.
  - На `TrapHitbox.Touched` (только когда trap.Active=true) → `dropAll(player)` через BindableEvent.
  - Цикл активации: `task.spawn` для каждой ловушки, чередует `Active=true/false` по `Period`/`ActiveDuration`.
- **`BrainrotEvents.TrapHit`** (BindableEvent в ServerStorage) — Slot для интеграции с Pickup без require'ов.

## Граничные случаи

- **Игрок не несёт брейнрота** — ловушка срабатывает, dropTop возвращает nil — без эффекта. Норма.
- **Игрок только что подобрал и сразу попал в ловушку** — debounce даёт 1.5с окна; за это время игрок отбежит дальше шипа. Норма.
- **Несколько игроков на одном хитбоксе** — каждый получает drop независимо (debounce per-player, не per-trap).
- **Игрок умирает в ловушке от другой причины (NPC?)** — `destroyAllCarried` по `hum.Died` уже срабатывает. TrapHit не нужен.
- **Trap-модель удалена в рантайме** — `MazeTrapsServer` слушает `GetInstanceRemovedSignal` и убивает свой `task.spawn` цикл (через флаг).
- **Игрок проходит через хитбокс пока он `Active=false`** — Touched не фильтруется по active, надо проверять явно: `if not trap:GetAttribute("Active") then return end` в callback'е.

Задача закрыта, когда:

2. ✅ В `Rare Maze` стоят минимум 3 шипо-ловушки (визуально видны, циклятся on/off).

6. ✅ В `common maze` (тир 1) **нет** ни одной ловушки.
7. ✅ Дебаунс работает — пробежка туда-обратно через одну и ту же ловушку за 1.5с роняет один раз, не три.
8. ✅ Ловушка после срабатывания не убивает игрока (`Health` остаётся ≥1).

## Что вне скоупа

- Уникальные эффекты ловушек (заморозка, сон, телепорт). Только drop.
- Балансировка density / period для всех 5 тиров — настраивается после плейтеста, в спеке только базовые цифры.
- Анимации шипов (плавный выезд / уход) — для MVP мгновенное появление через `Transparency` / `CanCollide`.
- Предупреждающий мигающий индикатор за 0.5с до срабатывания — nice-to-have, не блокер.
