# Метка владельца над базой

Игрок бежит из лабиринта с брейнротом и должен сходу понимать, какая из четырёх баз — его. Сейчас базы визуально неразличимы, номер `bazapl1..4` ничего не говорит, владельцы между сессиями меняются.

Спека: [Game Design/Base.md → Визуальное различение баз](../../Game%20Design/Base.md#визуальное-различение-баз).

## Что сделать

- Новый скрипт `src/ServerScriptService/BaseOwnerLabel.server.lua`.
- Слушает изменение атрибута `Owner` на каждой из `bazapl1..4`.
- Создаёт `BillboardGui` над базой с `DisplayName` владельца.
- Текст окрашен в персональный цвет игрока (детерминированная палитра по `UserId` или индексу слота — главное чтоб у разных игроков на сессии цвета не совпадали).
- `AlwaysOnTop = true`, `MaxDistance` ~ размер карты — чтоб видно было от выхода из лабиринта.
- Если `Owner == nil` (база свободна) — метка скрыта или показывает «свободно».

## Nice-to-have (отдельной задачей, не блокер)

- На HUD-е игрока пока он несёт брейнрота — подсвечивается ник/цвет его базы (или стрелка-указатель). Это уже про навигацию, делаем после базовой метки.

## Как тестить (для сына)

**Что проверяем:** видно ли над твоей базой твой ник, и не путается ли он с базами других игроков.

1. Зайди в игру **один**. Над одной из 4 баз должна быть твоя ник-табличка цветным.
2. Принеси брейнрота — увидишь свою табличку **из лабиринта** (от выхода)? ✅ — ОК.
3. Если есть второй игрок (или Studio с двумя плеями) — над его базой свой ник, **другого цвета**, не путается с твоим.
4. Выйди из игры → табличка над базой исчезла (или написано «свободно»).

### Что писать папе если что-то не так

- «Над моей базой ничего не висит» — метка не появилась.
- «Над моей и чужой базой одинаковый цвет» — палитра кривая, цвета совпали.
- «Видно только если стоять рядом» — `MaxDistance` слишком маленький.
