# Разбить большие скрипты на модули

`BrainrotPickup` (~205 строк) делает: input handling, weld rig setup, internal welds, drop logic, character lifecycle, prompt binding. `BrainrotDelivery` (~228 строк): leaderstats, ownership, placement, income tick, base touched binding. Single responsibility violation.

Разбить на ModuleScripts по responsibility:
- `WeldRig` — setup сварок брейнрот↔голова
- `PickupController` — обработка триггера, координация state
- `DeliveryController` — обработка touch, координация placement
- `IncomeTicker` — независимый цикл начисления

Server-скрипты становятся тонкими стартерами, соединяющими модули. Заодно появится возможность писать unit-тесты на модули (без Roblox API).
