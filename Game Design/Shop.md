# Магазин

Точка, где игрок тратит заработанные монеты на апгрейды и разблокировку лабиринтов. Это второе звено основного цикла после фарма ([Main Game Design.md](Main Game Design.md)).

## Доступ

- **Кнопка "🛒 Магазин"** — постоянно видна в правом верхнем углу экрана, под `leaderstats.Coins`.
- **Клавиша `B`** — открывает/закрывает магазин (одинаково).
- Магазин **доступен везде**: на базе, в лабиринте, на spawn-platform. Игрок может купить апгрейд хоть посреди рейда.

## Структура UI

```
┌──────────────────────────────────┐
│ 🛒 МАГАЗИН АПГРЕЙДОВ        ✕    │
├──────────────────────────────────┤
│ 💰 1234                          │
├──────────────────────────────────┤
│ 🏃 Speed              [Buy      ]│
│ How fast your character moves    │
│ lvl: 18 → 20            💰 300   │
├──────────────────────────────────┤
│ 📦 Max Carry          [Buy      ]│
│ Brainrots you can carry at once  │
│ lvl: 1 → 2              💰 250   │
├──────────────────────────────────┤
│ 🏠 Base Slots         [Buy +1   ]│
│ +1 placement slot on your base   │
│ Slots: 14 / 30          💰 500   │
└──────────────────────────────────┘
```

Трек "Base Slots" по UX-семантике = "купи слот", не "купи уровень". Серверная сторона по-прежнему хранит `baseLvl` (1..21), но в UI это скрыто — показываем лишь текущее значение `Capacity` ("Slots: X / 30") и "+1 slot" на кнопке.

## Треки апгрейдов

3 трека. Все цены и значения — в [Economy.md](Economy.md):

| Иконка | Трек | Что меняет | Уровни |
|---|---|---|---|
| 🏃 | Скорость | `Humanoid.WalkSpeed` персонажа | 1–10 (16 → 70) |
| 📦 | Переноска | Сколько брейнротов несёшь стопкой над головой | 1–8 (1 → 8) |
| 🏠 | База | +1 слот за покупку (500 монет за слот). Каждые 10 слотов — новый этаж | 1–21 (10 → 30 слотов, см. [Base.md](Base.md)) |

## UX правила

1. **Эффект применяется мгновенно** при покупке. Скорость — на ближайший шаг. Переноска — на следующий пикап. База — атрибут `Capacity` обновляется сразу, при следующей доставке учитывается.
2. **Цвета кнопок:**
   - 🟢 **Зелёная** — хватает монет, готов купить.
   - 🔴 **Красная** (неактивная) — не хватает монет (показывает сколько надо).
   - ⚫ **Серая "МАКС"** — достигнут максимум, кнопка отключена.
3. **Live-обновление:** пока магазин открыт, при тике дохода Coins-счётчик обновляется в шапке, цвета кнопок переключаются (красная → зелёная при накоплении нужной суммы).
4. **Без подтверждения** покупки. Один клик = транзакция. (Если позже окажется что игроки промахиваются — добавим dialog confirm.)

## Что НЕ в Магазине (отдельные системы)

- **Разблокировка лабиринтов** — отдельный экран / NPC у телепорта (см. [Labyrinths.md](Labyrinths.md), цены в [Economy.md § 5](Economy.md)). По дизайну это "глобальная" покупка, не "апгрейд персонажа", поэтому разнесено.
- **Скины / косметика** — Robux-only, отдельный premium-shop. Не для MVP.
- **Энергия / временные бусты** — не в дизайне.

## Технические артефакты

- `ServerScriptService.UpgradeService` — покупка + применение эффектов.
- `ReplicatedStorage.UpgradeConfig` — таблицы значений и цен.
- `ReplicatedStorage.PurchaseUpgrade` (RemoteEvent) — клиент → сервер.
- `ServerStorage.PlayerData` — текущие уровни (in-memory, future: DataStore).
- `StarterPlayerScripts.ShopClient` — UI.

## Идеи на будущее (не для MVP)

- **Анимация при покупке** — coin-ливень, звук cha-ching.
- **Tooltip** на hover — что конкретно даст этот уровень.
- **Sticky highlight** на доступных к покупке (зелёных) — пульсирующий бордер.
- **Bulk-buy** — "купить +5 уровней" одной кнопкой за суммарную цену.
- **История покупок** для аналитики (что апгрейдят первым, какие застревают).
- **Sale events** — −20% на трек скорости в выходные.
