# User Session (сохранение прогресса)

Игра должна сохраняться для каждого игрока — игрок может выйти и продолжить с того же места. Сейчас весь прогресс теряется при закрытии игры (см. [`PlayerData.lua`](../src/ServerStorage/PlayerData.lua) — комментарий «in-memory, future: backed by DataStore»).

## Ключевое правило: оффлайн = пауза

**Брейнроты НЕ зарабатывают деньги, когда игрок вне игры.** Доход капает только пока игрок в игре. После выхода тиканье останавливается, при возврате — продолжается с того же значения коинов.

Зачем: игра поощряет активную игру, а не AFK-фарминг. Иначе теряется смысл проходить лабиринты — выгоднее зайти, наставить брейнротов, выйти на сутки.

В коде ([`BrainrotDelivery.server.lua`](../src/ServerScriptService/BrainrotDelivery.server.lua) `task.spawn` с `task.wait(1)`) тик и так работает только по онлайн-игрокам — никаких изменений не нужно. Просто **не** реализовывать «offline accumulation».

## Что сохраняем


| Поле          | Источник                                                | Зачем                                                                                         |
| ----------------- | --------------------------------------------------------------- | -------------------------------------------------------------------------------------------------- |
| `coins`           | `leaderstats.Coins`                                             | Валюта на покупки в[Shop](Shop.md)                                                 |
| `speedLvl`        | `PlayerData.speedLvl`                                           | Уровень апгрейда скорости                                                   |
| `carryLvl`        | `PlayerData.carryLvl`                                           | Уровень апгрейда вместимости рук                                      |
| `baseLvl`         | `PlayerData.baseLvl`                                            | Уровень апгрейда базы                                                           |
| `placedBrainrots` | список `{name, cframe}` для каждого брейнрота на базе | Чтобы при возврате брейнроты были на тех же местах, где игрок их поставил |

## Что НЕ сохраняем

- **Брейнроты в руках** — теряются при выходе (как и при смерти, см. [Main Game Design § Взятие брейнрота](Main%20Game%20Design.md)).
- **Позиция персонажа** — после захода игрок появляется на базе.
- **Открытость магазина и любые UI-состояния** — клиентская временная штука.
- **Активные таймеры спавн-пэдов** — лабиринт общий, не персональный.
- **Накопленный оффлайн-доход** — см. правило выше.

## Когда сохраняем

1. **`PlayerRemoving`** — основной триггер. Игрок вышел → сразу сейв.
2. **`game:BindToClose`** — сервер закрывается → сейв всем оставшимся игрокам (с увеличенным timeout, иначе DataStore не успеет).
3. **Периодический автосейв** — каждые 3 минуты. Защищает от потери при крэше сервера.

## Когда загружаем

- **`PlayerAdded`** — однократно при заходе. До загрузки игрок не должен ходить (заморозить движение / показать loading-overlay), иначе можно успеть купить апгрейд на дефолтных коинах.
- **При ошибке DataStore** — не давать пустой профиль (это сотрёт реальные данные при следующем сейве). Лучше **кикать с сообщением «Не удалось загрузить, попробуй снова»**.

## Технические детали

- **DataStore name:** `PlayerData_v1` (версия в имени — на случай миграции схемы).
- **Key:** `player_<UserId>` (`player_123456789`).
- **Формат:** обычная Lua-table, Roblox сам сериализует.
- **Schema versioning:** добавить `schemaVersion = 1` в сохраняемую таблицу. При загрузке: если версия меньше — мигрировать.
- **Retry при сбое DataStore:** 3 попытки с экспоненциальной задержкой (1с, 2с, 4с).

## Edge cases

- **Игрок зашёл, до загрузки данных вышел** — отменяем сейв с дефолтными значениями (защита от затирания).
- **Базы `bazapl1..4` назначаются по слотам, не персонально** — при возврате игрок может попасть на другой слот. Сохранение `placedBrainrots` должно быть привязано **к UserId, а не к имени базы** (`bazapl1` vs `bazapl3`).
- **Брейнрот удалили из спеки** ([Brainrots.md](Brainrots.md)) — при загрузке его модель не найдётся в `BrainrotTemplates`. Решение: пропустить + лог `[Session] skipped unknown brainrot 'X' for player Y`.
- **Апгрейд удалили из [Shop.md](Shop.md)** — уже потраченные коины **не возвращаем**. Уровень в `PlayerData` остаётся, но если апгрейда больше нет в [`UpgradeConfig`](../src/ReplicatedStorage/UpgradeConfig.lua) — эффекта тоже нет.
- **Игрок в двух местах одновременно** — Roblox single-server гарантирует уникальность, но если когда-то добавим teleport между серверами — нужно session lock.
