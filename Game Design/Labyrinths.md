# Лабиринты

## 1. Уровни сложности

| № | Уровень | Особенности |
|---|---|---|
| 1 | Обычный | Простые редкие шипы (5 шт, медленные — обучающие) |
| 2 | Редкий | Шипы (5 шт, базовая скорость) |
| 3 | Эпичный | Шипы (5) + огненные струи (3) + проваливающийся пол (2), скорость +20% |
| 4 | Легендарный | Шипы (8) + огонь (5) + пол (3) + падающая наковальня (2), скорость +40% |
| 5 | Мифический | Шипы (12) + огонь (8) + пол (4) + наковальня (3) + сужающиеся стены (2), скорость +60% |
| 6 | Божественный | — (WIP) |

- Чем выше сложность лабиринта, тем он больше и тем более редкие и прибыльные брейнроты в нём появляются.
- С каждым уровнем количество и сложность ловушек растёт.

## 2. Структура лабиринта

- Лабиринт **фиксированный** — расположение коридоров и ловушек одинаковое каждый раз. Игроки могут запоминать маршруты.

## 3. Площадки с брейнротами

- Брейнроты появляются автоматически, интервал — от 30 до 90 секунд.
- На каждой площадке появляется случайный брейнрот:
  - В начальных лабиринтах — из обычных редкостей.
  - В более сложных — более высоких редкостей.
- Полный список брейнротов — в [Brainrots.md](Brainrots.md).

## 4. Ловушки

- Появляются с уровня 1 (Обычный) — простые медленные шипы как обучение.
- **Эффект при срабатывании единый для всех типов:** игрок роняет всех брейнротов, которых нёс. Они падают перед ним на пол, их можно подобрать снова (в т.ч. другому игроку).
- **Не убивает** — только роняет добычу. HP игрока не меняется.
- Период срабатывания зависит от тира лабиринта: Tier 1 = ×1.5 (медленно), Tier 2 = базовая, дальше +20%/тир.
- Дебаунс 1.5 секунды per-player — пробежка туда-обратно через одну ловушку за 1.5с роняет один раз, а не дважды.

### Типы ловушек

| Тип | Где | Триггер | Реализован |
|---|---|---|---|
| Шипы из пола | T1+ | Periodic | ✅ |
| Огненные струи из стен | T3+ | Periodic | ✅ |
| Проваливающийся пол | T3+ | Reactive (Touched) | ⏳ |
| Падающая наковальня | T4+ | Periodic | ⏳ |
| Сужающиеся стены | T5+ | Periodic | ⏳ |

Реализация: [`MazeTrapsServer`](../src/ServerScriptService/MazeTrapsServer.server.lua) + `ServerStorage.TrapTemplates.{SpikeTrap, FireJetTrap, …}`.

### 4.1. Шипы из пола (`Spikes`) — реализовано

Wedge-парты, торчащие из плитки. В Idle серые `Metal`, transparency 0.4. В Active — яркий `Neon`, цвет `Color3.fromRGB(255, 60, 60)`. Hitbox — отдельный Part над шипами с тегом `TrapHitbox`, `CanTouch` следует за `Active`.

- **Цикл.** Periodic. `Period 3s / ActiveDuration 1s`.
- **Где.** Узкие коридоры, T1+. На T1 период удваивается (×1.5) для обучения.

### 4.2. Огненные струи из стен (`FireJet`) — реализовано

`Beam` с градиентом красно-оранжевого + `ParticleEmitter`, бьют поперёк коридора. `Beam.Enabled` и `ParticleEmitter.Enabled` следуют за `Active`. Hitbox — Part 1×1×6 в зоне струи.

- **Цикл.** Periodic. `Period 2.5s / ActiveDuration 0.8s`.
- **Где.** Открытые проходы и Т-образные перекрёстки, T3+.

### 4.3. Проваливающийся пол (`FloorDrop`) — TODO

- **Визуал.** Плитка пола 4×0.5×4, в Idle серая с трещиной (decal или текстура «WARNING»). При активации трескается (частицы пыли + звук «хруст»), потом проваливается — `Transparency = 1, CanCollide = false`. Через `RestoreDelay` секунд возвращается обратно в исходное состояние.
- **Триггер reactive, не periodic.** Игрок наступает → telegraph `WarmupDelay` (плитка трещит, hitbox ещё активен) → проваливается на `RestoreDelay` → восстанавливается. Других триггеров на этой плитке в этот момент нет (debounce per-trap).
- **Hit.** В момент перехода в провалившееся состояние любой стоящий на плитке игрок получает `TrapHit` (упадёт вниз сам по физике, эффект — drop брейнротов).
- **Атрибуты модели.** `WarmupDelay = 0.4`, `RestoreDelay = 2`. `Period`/`ActiveDuration` не используются.
- **Где.** Открытые залы лабиринта, T3+.

### 4.4. Падающая с неба наковальня (`Anvil`) — TODO

- **Визуал.** Тёмный куб/меш, `Material = Metal`, тяжёлый. В Idle висит выше потолка лабиринта (вне видимости игрока) или скрыт. За `WarmupDelay` секунд до удара на пол под точкой падения кладётся круглая чёрная «тень» (Decal на полу или Part с `Transparency = 0.5`) — telegraph для игрока.
- **Цикл.** Periodic. Раз в `Period` секунд: куб телепортируется к точке падения → telegraph (тень) → падает с `DropHeight` → лежит на полу `LandingDelay` → исчезает (или возвращается наверх). Точка падения **фиксированная** (атрибут на trap-модели), не следит за игроком — иначе непроходимо.
- **Hit.** Hitbox — сам куб. Активен во время падения и пока лежит.
- **Атрибуты модели.** `Period = 4`, `WarmupDelay = 0.5`, `DropHeight = 30`, `LandingDelay = 1`.
- **Где.** Залы с высоким потолком, T4+.

### 4.5. Сужающиеся стены (`CrushingWalls`) — TODO

- **Визуал.** Две параллельные стены, в Idle разнесены на `OpenWidth ≈ 8` студов (свободный проход). В Active съезжаются на `ClosedGap ≈ 1` (узкая щель — и хитбоксы стен встречаются в центре). Tween между Open и Closed, не мгновенно.
- **Цикл.** Periodic. `Period = OpenDuration + SqueezeDuration + HoldDuration + ReleaseDuration`. Например: 2.5 + 1.2 + 0.8 + 0.5 = 5с.
- **Hit.** Hitbox — внутренние поверхности стен (тег `TrapHitbox`). Фаерят `TrapHit` в `HoldDuration` (когда стены сошлись и держат).
- **Атрибуты модели.** `OpenWidth = 8`, `ClosedGap = 1`, `SqueezeDuration = 1.2`, `HoldDuration = 0.8`, `OpenDuration = 2.5`, `ReleaseDuration = 0.5`.
- **Где.** Длинные прямые коридоры, T5+.
