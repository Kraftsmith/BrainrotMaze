# Multiplayer

Игра рассчитана на нескольких игроков в одном мире одновременно. Это **кооперативно-параллельный** режим: общий мир, своя прогрессия у каждого, минимальная прямая конкуренция. Без PvP-урона.

## Сколько игроков

- **До 4 игроков на сервер** — ограничено числом баз `bazapl1..4` (см. [`BaseManager.server.lua`](../src/ServerScriptService/BaseManager.server.lua)).
- При попытке зайти 5-му игроку — `TeleportService:TeleportAsync` отправляет на другой инстанс той же игры. Если телепорт упал — кикает с сообщением «Сервер пустой! Попробуй снова».

## Что общее, что персональное

| Элемент | Общее / Персональное |
|---|---|
| Лабиринты (common / rare / epic / ...) | **Общие** — все ходят по одним и тем же |
| Spawn platforms (1..6) и пэды на них | **Общие** — кто первый добежит, тот и забрал |
| Брейнроты на пэдах | **Общие** — race-protected (см. ниже) |
| Брейнроты в руках | Персональные — у каждого своя «башенка над головой» |
| База `bazapl_` | Персональная — назначается слотом, прокачивается отдельно |
| Брейнроты на базе | Персональные, доход капает только владельцу |
| Коины, апгрейды | Персональные, сейв через [User Session](User%20Session.md) |
| Магазин | Открывается у каждого свой (UI клиента) |
| START кнопка / телепорт в лабиринт | Общий механизм, у каждого свой кулдаун (по факту — никакого кулдауна) |

## Конкуренция за брейнротов

### На пэде (свежий спавн)
- Все игроки видят одного и того же брейнрота на пэде.
- Любой может нажать `E` и взять.
- **Race protection:** атрибут `BeingPickedUp` на модели (см. [`BrainrotPickup.server.lua`](../src/ServerScriptService/BrainrotPickup.server.lua) `pickup` функция). Кто первым успел в один тик — тот забрал, остальные получают молчаливый отказ.

### Уронил брейнрота (трапы / `E` в руки)
- Брейнрот лежит на земле с включённым `ProximityPrompt`.
- **Подобрать может ЛЮБОЙ игрок**, не только уронивший.
- Это создаёт пространство для лёгкого «грифинга»: пробежал мимо чужого упавшего брейнрота, забрал — он твой. Дизайн-решение в пользу динамики, не защиты.

### На чужой базе
- Чужие базы видны издалека (см. [Main Game Design § База](Main%20Game%20Design.md)).
- Подойти к чужой базе можно физически, но клавиша `E` не сработает — `BrainrotDelivery` проверяет `base:GetAttribute("Owner") == player.Name` (см. [Base.md § Размещение брейнротов](Base.md)). Если ты не владелец — E на чужой базе делает обычный дроп брейнрота на пол перед игроком.
- Поднять чужого размещённого брейнрота нельзя — у placed-брейнротов `ProximityPrompt.Enabled = false`.

## PvP

- **Урон выключен.** Игроки не могут убивать друг друга.
- Лабиринтные ловушки наносят урон только пробегающему — если другой игрок рядом стоит, его не задевает (per-character touch).

## Коммуникация

- Стандартный Roblox чат (`TextChatService`) — глобальный для сервера.
- **Без proximity-чата.**
- Команды (`/upgrade base`, `/vip on`) — каждый игрок может вводить только за себя.

## Сейв и переход между серверами

- Прогресс игрока **привязан к UserId**, не к серверу. См. [User Session](User%20Session.md).
- Зашёл в любой инстанс игры → загрузился твой профиль → играешь со своим прогрессом.
- При выходе сейв пушится в DataStore. На любом другом сервере при следующем заходе подтянется.

## Edge cases / открытые вопросы

- **Игрок забрал брейнрота и вышел из игры с ним в руках.** По дизайну: брейнрот теряется (см. [Main Game Design § Взятие брейнрота](Main%20Game%20Design.md)). Не возвращается на пэд — просто исчезает.
- **Все 4 базы заняты, все 4 игрока вышли.** Базы освобождаются по `PlayerRemoving` (`baseOccupied[baseName] = false`). Следующий зашедший получит первую свободную.
- **Игрок зашёл, его привычная база `bazapl1` занята другим — попал на `bazapl3`.** Сейв `placedBrainrots` хранит относительные позиции (см. [User Session § Edge cases](User%20Session.md)), так что брейнроты восстановятся в новой базе на тех же местах.
- **Telesponge-троллинг** (один игрок постоянно блокирует чужого у телепорта) — не решено в коде. Пока полагаемся на правила сервера / репорты Roblox.

## Открытые вопросы

- **Ивенты для всех игроков на сервере одновременно** (типа «золотой брейнрот в Mythic-лабиринте на 30 секунд»). Заявка на отдельный документ.
- **Совместная статистика сервера** (сколько брейнротов пойманно за сессию всеми) — пока не нужна.
- **Дружбы / приглашения играть в один сервер** — Roblox даёт `JoinSuccess` через социальные функции, отдельной механики не пишем.
